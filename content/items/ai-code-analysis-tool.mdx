This project is still being shaped, so the public write-up is intentionally lightweight.

The working direction is a command-line tool that can inspect a repository, extract useful structure, and help a developer ask better questions about the codebase. The interesting part is not simply calling an LLM; it is deciding what context to collect, how to keep that context grounded, and how to present results in a way that supports engineering judgment instead of replacing it.
